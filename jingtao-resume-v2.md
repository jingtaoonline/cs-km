#### 主导小团 Agent2.0 上下文工程重构：Token 消耗降低 70%（2025.12 - 2026.4）

**背景问题**：小团 Agent 接入美团 10+ 垂类业务后，Prompt 和业务知识快速膨胀，单轮请求存在大量无关Prompt；高峰 QPS 扩大 10 倍以上后，推理资源消耗、模型回复质量、和业务迭代效率成为主要瓶颈。

**核心工作**：

1. 主导设计 Skill 渐进式披露链路，将 Skill 拆为 `summary / activation condition / full instruction / tool schema` 等信息层级；首轮上下文仅注入摘要+路由信息，模型命中意图后再通过 tool call 拉取完整 Skill 内容。
2. 重构上下文构建链路，废弃大而全 Prompt，将上下文拆成基础设定、业务自定义、技能区和对话区；基于 FreeMarker 模板按业务、模型、实验分组、Skill 版本动态渲染，支持不同模型和灰度实验下快速切换上下文策略。
3. 参与建设入口网关意图分流能力，结合规则匹配和小模型/SLM 判断请求类型；简单意图绕过多步 Agent Tool/Workflow 推理，直接走传统搜索 + 单次大模型总结链路，复杂意图提前预加载相关 Skill。
4. 落地短期记忆压缩机制，基于 context 使用量阈值触发异步压缩和滚动摘要；按可重建性优先裁剪中间推理过程、搜索结果和工具执行结果，将历史多轮对话压缩为结构化摘要，保留当前目标、已确认结果、拒绝选项、偏好约束和待处理事项。
5. 参与长期记忆提取与注入链路设计，会话结束后异步提取结构化偏好和语义偏好；结构化信息通过槽位存储，语义偏好提炼为自然语言片段并向量化存储，会话开始时结合意图识别和当前 query 检索相关记忆，并控制注入 token 上限。
6. 参与建设知识库接入链路，支持文档清洗、分片、预览、索引入库和召回调试；实现结构化数据离线 pipeline，定时调度商家、供给、运营配置等数据导入索引库，减少业务知识手工维护成本。

**结果产出**：Agent2.0 全量上线后，整体 Token 消耗降低 70%，业务需求迭代效率提升 30%，badcase 率降低 5%，用户 7 天留存率提升 8%。

#### 需求智能分析助手（AI Coding Agent）｜2024.6 - 2025.10

项目核心开发者，全流程参与从需求分析到代码交付的 Agent 自动化链路设计开发。

**背景问题**：美团首页推荐团队作为 App 的流量入口，日常需求迭代压力大，传统研发流程效率低且依赖人工经验，通过 AI 技术实现从需求分析到代码交付的自动化提效。

**核心工作**：

1. 参与设计双模式需求处理引擎，针对模板化需求建设 preset workflow 模式，命中模板后从知识库精确拉取该类需求的背景知识、代码模板和示例，自动生成技术方案并按类级粒度拆分子任务执行改动；针对个性化需求建设 ReAct 闭环模式，召回业务知识，结合现有代码主动向用户提问澄清，循环收敛后生成技术方案。
2. 建设人工审核节点，在技术方案生成后、PR 生成后两个关键节点设置人工 Review，确保自动生成内容在进入下一阶段前经过确认，兼顾自动化效率与代码质量。
3. 主导建设分层知识库系统，解决模型不理解业务的问题，知识来源为代码库自动化扫描和人工经验沉淀。模板类需求走精确匹配，命中后直接注入完整的背景知识+代码示例；文档类知识走 BM25 / 向量 粗召回 + 大模型语义精排的两阶段检索。
4. 参与通用工具建设，实现本地代码检索工具、网络搜索工具、Bash 执行工具、远程代码仓库查询工具等。
5. 主导搭建自动化测试与发布链路，覆盖单元测试和端到端集成测试，流水线自动部署测试环境后运行单测、回放测试集。
6. 参与双端产品形态实现，管理端（Web）支撑知识库维护、模板配置、测试集管理和任务管理等；IDEA 插件端承载对话交互、代码检索和主流程执行。

**结果产出**：模板化需求交付时间缩短到 0.5PD 以内；个性化需求效率提升 30% 以上；代码采用率达到 80% 以上。

#### 推荐链路性能优化与工程治理（2024.10 - 2025.8）

1. 推动首页推荐埋点治理，针对近 500 个历史埋点字段建设 Java 反射访问监控，识别字段使用情况和下游依赖；按裁剪字段、一般字段和重点字段分级治理，一般字段改为写 Kafka 到 Hive，降低主链路返回体积。
2. 负责渲染信息预调用优化，在直播、特团召回完成后由聚合服务异步推送候选结果给渲染服务，提前调用渲染信息接口并写入 Guava 本地缓存；重点处理异步时序、缓存 key、有效期和未命中兜底。
3. 建设请求 Mock 与调试回放能力，支持请求日志采集、gzip 压缩、traceId 回放、实验加白、debug 信息透出和强插供给，将 badcase 排查流程沉淀为多团队复用工具。
4. 参与首页推荐渲染服务技术重构，将渲染服务拆分为 IO、渲染算子、数据映射 3 层，设计统一路由映射、DAG 执行图、字段粒度算子和注解约束机制。

**结果产出**：推荐接口端到端耗时分别降低 140ms / 40ms，badcase 排查从小时级降低到分钟级，渲染服务重构后提升服务稳定性和可维护性。
