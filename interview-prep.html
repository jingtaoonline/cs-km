小团 Agent2.0 上下文工程重构核心项目
2025.12 - 2026.4 ｜ Token 消耗降低 70%，用户 7 天留存提升 8%
渐进式披露 Skill 架构
渐进式披露（Progressive Disclosure）的核心思路是什么？你们是怎么判断哪些 Skill 需要预加载、哪些只注入 summary？
Skill summary 的内容是如何设计的？如果 summary 信息不足，模型会不会激活错误的 Skill？你们是怎么评估召回准确率的？
按需激活的 tool call 链路上有额外延迟，用户侧能感知到吗？你们是如何在延迟和 Token 消耗之间做权衡的？
当用户意图跨多个垂类时，会同时激活多个 Skill 吗？这时候上下文管理如何处理？
考察方向：Skill 路由设计、召回精度评估、多轮对话下的 Skill 生命周期管理
你提到用 FreeMarker 模板动态渲染上下文，模板管理是怎么做版本控制和灰度的？
不同模型（大模型/SLM）对同一套 Prompt 模板的效果差异怎么处理？有没有遇到模板迁移导致效果劣化的情况？
模板参数是实验系统动态下发的，如果实验参数和模板字段不匹配，有什么防护机制？
意图分流与入口网关
入口网关的意图分流用了规则 + 小模型/SLM，这两者的分工是什么？规则覆盖哪些场景，SLM 补哪些？
SLM 的分类效果是怎么衡量的？如果 SLM 判错导致走了错误路径，有什么兜底机制？
简单意图走"传统搜索 + 大模型总结"，这条链路相比 Agent 全流程的体验差异用户能接受吗？你们怎么定义"简单意图"的边界？
随着业务迭代，意图类别会增加，规则维护成本会不会越来越高？有没有自动化的规则生成或更新机制？
记忆压缩与知识库
记忆压缩里"规则裁剪工具调用明细"是怎么做的？你们的裁剪规则是什么，有没有因为裁剪丢失关键上下文导致模型答错的 case？
全局摘要是实时生成还是按轮次更新的？摘要本身的质量怎么保证，有没有摘要信息失真的问题？
长期记忆是通过"标签配置 + 离线任务"提取的，提取准确率怎么样？用户隐私和记忆存储有没有做过专门的合规设计？
知识库链路里，文档分片策略是怎么设计的？分片粒度对检索效果有什么影响？
结构化数据离线 pipeline 和向量检索的文档是统一索引还是分开索引？检索时怎么融合结构化和非结构化的结果？
知识库的更新频率是多少？增量更新时如何避免索引不一致？
多级缓存与结果指标
多级缓存基于"query 相似度 + 地理位置"，query 相似度是怎么计算的？阈值怎么定？缓存命中错误的 case 怎么处理？
缓存失效策略是什么？供给信息实时性要求高的场景（比如商家营业状态），怎么保证缓存不影响准确性？
Token 降低 70% 是怎么测量的？是单次对话的平均 Token 数，还是某个时间周期的总量对比？有没有 Token 减少但质量下降的情况？
